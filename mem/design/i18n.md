---
name: i18n architecture
description: How the bilingual (Persian/English) setup is wired in this project
type: design
---
- Library: `i18next` + `react-i18next` + `i18next-browser-languagedetector`.
- Config: `src/i18n/index.ts` (initialized once, idempotent guard).
- Locale files: `src/i18n/locales/fa.json` and `src/i18n/locales/en.json`. Keys must mirror each other exactly.
- Default/fallback language: `fa`. Persisted in `localStorage` key `president_lang`.
- `applyHtmlDir(lng)` in `src/i18n/index.ts` sets `<html lang dir>` on every `languageChanged` event.
- `src/routes/__root.tsx` imports `../i18n` and updates `<html>` attributes via `useEffect` on `i18n.language`.
- `LanguageToggle` button is placed in the auth screen and the top nav header inside `src/App.tsx`.
- Status: phase 1 covers auth screen, header, tabs, `RoleTable`, `PlayerCard`, and all role names/abilities. Phase 2 needs to translate: `Day0Setup`, `NightWizard`, `Night0Terrorist`, `ChaosPhase`, `GameGuide`, `CollapsibleGuide`, `ConditionsModal`, `SearchManager`, `SpeakingTimer`, the rest of `App.tsx` (tab labels, cabinet HUD, day-phase wizard, gameover modals, log messages).
